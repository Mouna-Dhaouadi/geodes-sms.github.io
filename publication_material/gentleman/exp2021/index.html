Modelling with a projectional editor

Jetbrains MPS and Gentleman

  • PDF version

In this experiment, you will accomplish several modelling tasks using the projectional editors Gentleman and Jetbrains MPS.

Experiment

In the following tasks, we will create, manipulate and evaluate traffic light (TL) configurations.
After the tasks, we will take a short survey to gather some additional feedback regarding your experience.

Task 1: Create a TL model

Create a TL model with the following three lights: Go (color) as GREEN (name), Hurry as YELLOW and Stop as RED.

  • Stop behaviours
    • [temporal: after 15 s] --> Go
  • Hurry behaviours
    • [temporal: after 5 s] --> Stop
  • Go behaviours
    • [temporal: after 30 s] --> Hurry

Create the corresponding TL model. [Estimated time: 5 minutes]

  • Gentleman performance (data)
  • MPS performance (data)
  • Results

Task 2: Edit a model

Add the two following modes: Normal and Police

Mode Normal

  • Lights: same as those created in task 1
  • Initial light (start): Stop
  • Behaviours
    • [manual: On signal] --> Police

Mode Police

  • Lights: ON as YELLOW and OFF as BLACK
    • ON behaviours
      • [temporal: after 2 s] --> OFF
    • OFF behaviours
      • [temporal: after 2 s] --> OFF
  • Initial light (start): ON
  • Behaviours
    • [manual: On signal] --> Normal

Change the created TL model to respect the new requirements. [Estimated time: 5 minutes]

  • Gentleman performance (data)
  • MPS performance (data)
  • Results

Task 3: Evaluate a model

A friend needs your expertise to evaluate a TL model. He wants to create an automatic mode with the following requirements

  • Lights: GL as GREEN, YL as YELLOW and RL as RED
  • Initial light: GL
  • Allowed behaviours:
    • GL --> YL
    • YL --> RL
    • RL --> GL
  • No manual behaviours

1. Identify the errors found in the model.
2. Apply the changes to respect the requirements.
[Estimated time: 7 minutes]
